# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Purpose

This is the `abebars/abebars` GitHub profile repository. The `README.md` at the root is rendered as Ahmed Bebars's public GitHub profile page. There is no build system, package manager, or test suite — all content is static Markdown, SVG, images, and a few JS/CSS assets.

## Key Files and Their Roles

- **`README.md`** — The GitHub profile page. Edits here are immediately visible on the profile after pushing to `main`.
- **`talks/README.md`** — Running history of conference talks, podcasts, and community organizing events. Both "Up Next" and "Past Talks" tables are maintained here.
- **`github-metrics.svg`** — Auto-generated by the GitHub Actions workflow; do not edit manually.
- **`.github/workflows/profile.yml`** — Runs `lowlighter/metrics@latest` twice daily (00:30 and 12:30 UTC) and on every push to `main`. It requires a repository secret `METRICS_TOKEN` (a GitHub PAT with appropriate scopes).
- **`assets/`** — Static files referenced from `README.md` or the personal website (`abebars.io`). The `stack-share.js` embeds StackShare iframes; `extra.css` is a stub.

## Workflow

Changes are made directly to Markdown files and pushed to `main`. The metrics workflow runs automatically — there is no manual build step.

When updating `talks/README.md`, keep entries in reverse-chronological order within each table. The "Up Next" table moves to "Past Talks" after the event date passes.

Certification badge images live in `assets/images/` and are also hosted on `abebars.io`; both paths are referenced in `README.md`.
